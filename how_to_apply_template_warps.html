How to transform between various macaque MRI templates¶

Author: Nikoloz Sirmpilatze

Last updated: 11 Oct 2019

Requirements:

  1. Using the command line
    • ANTs >= 2.1.0
      • antsApplyTransforms needs to be in your path as executable
  2. Using the nipype interface
    • ANTs >= 2.1.0
      • antsApplyTransforms needs to be in your path as executable
    • python >= 3.7
    • nipype >= 1.2.0

PS: if you have older versions of ANTs or python on your system, make sure to still give it a try. There is a high chance that it will work.

Warps are provided for transformations between the following templates

  1. NMT
  2. D99
  3. INIA19
  4. MNI macaque
  5. Yerkes19

Within this notebook, they are abbreviated as NMT, D99, INIA, MNI, and YRK

There are 20 warps in total (10 unique template pairs, with 2 directions per pair). The naming of the warps follows the scheme:

  • InputAbbrev_to_OutputAbbrev_CompositeWarp.nii.gz

For example:

  • NMT_to_D99_CompositeWarp.nii.gz

The 'Composite' refers to the fact that this warp combines information from a linear affine transform and a non-linear SyN transform.

Example 1: Transforming an atlas (parcellation)¶

Let's take the D99 atlas and transform it from D99 space to MNI macaque space

In [52]:
plot_roi(D99_atlas, bg_img=D99_brain, annotate=False,
        title='D99 atlas on D99 template');

First let's define relative paths to the necessary files.

You need to change these depending on where they are stored on your system.

In [53]:
templates_path = '/home/nsirmpilatze/home@dpz/templates/'

D99_path = templates_path + 'D99/'
D99_brain = D99_path + 'D99_template.nii.gz'
D99_atlas = D99_path + 'D99_atlas_1.2a.nii.gz'

MNI_path = templates_path + 'MNI_macaque/'
MNI_brain = MNI_path + 'macaque_25_model-MNI_brain.nii.gz'
# The name of the output file
D99_atlas_in_MNI = templates_path + 'D99_atlas_1.2a_in_MNI.nii.gz'

# Full (affine + SyN) composite warp 
warp = templates_path + 'D99_to_MNI_CompositeWarp.nii.gz'

Option a: using antsApplyTransforms through the command line¶

Command:

antsApplyTransforms -i $D99_atlas -r $MNI_brain -o $D99_atlas_in_MNI 
-t $warp -n NearestNeighbor

Let's explain the arguments, which are passed through flags:

  • -i: input image
  • -r: reference image (target space)
  • -o: output image
  • -t: transform
  • -n: interpolation
    • The default is Linear, but here we set it to NearestNeighbor. Since the parcellation is defined with discrete numerical values, this ensures that we won't end up with any intermediate numbers in the output image

Let's give this a try

In [54]:
!antsApplyTransforms -i $D99_atlas -r $MNI_brain -o $D99_atlas_in_MNI -t $warp -n NearestNeighbor
In [55]:
plot_roi(D99_atlas_in_MNI, bg_img=MNI_brain, annotate=False, dim=-0.5,
        title='D99 atlas on MNI template');

Option b: using the nipype interface for antsApplyTransforms¶

Read the documentation here

In [56]:
apply_warp = ants.ApplyTransforms(
                input_image=D99_atlas,
                reference_image=MNI_brain,
                output_image=D99_atlas_in_MNI,
                transforms=warp,
                interpolation='NearestNeighbor')
apply_warp.run();
In [57]:
plot_roi(D99_atlas_in_MNI, bg_img=MNI_brain, annotate=False, dim=-0.5,
        title='D99 atlas on MNI template');

The results with both options are identical

NMT_path = templates_path + 'NMT/NMT_v1.2/'

stat_map_in_NMT = templates_path + 'example_stat_map_in_NMT.nii.gz'

Option a: using antsApplyTransforms through the command line¶

Notice that this time we use Linear interpolation. More advanced interpolation options are also available (e.g. 'BSpline').

In [60]:
!antsApplyTransforms -i $stat_map_in_NMT -r $D99_brain -o $stat_map_in_D99 -t $warp -n Linear
